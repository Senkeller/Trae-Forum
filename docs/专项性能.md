**1. 目标与范围**

1. 目标问题：移动端长列表（首页流、回复流、消息流、搜索结果）滑动偶发卡顿、掉帧。
2. 目标手段：懒构建 + 减少重绘与主线程负载 + 图片链路优化 + 滚动触底触发治理。
3. 目标页面：首页 Feed、AI 快讯、帖子详情回复、消息列表、消息详情、搜索结果、用户主页动态流。
4. 非目标：不改 API 协议、不改业务功能、不新增第三方依赖、不做 UI 风格大改。

**4. 开发任务拆解（按优先级）**

1. P0-1 新增滚动触底治理通用组件。\
   文件：新增 lib/core/utils/scroll\_load\_guard.dart。\
   要求：实现“触底阈值 + 最小触发间隔 + 请求中门闩”三件套。\
   要求：API 可被页面复用，形态建议 tryTrigger(ScrollController, Future\<void> Function())。\
   要求：默认阈值 200\~240px，默认最小间隔 180ms。\
   要求：并发请求禁止重复触发。
2. P0-2 将滚动触底页面全部接入统一治理。\
   文件：\
   feed\_detail\_page.dart\
   post\_reply\_list.dart\
   user\_profile\_page.dart\
   message\_page.dart\
   message\_detail\_page.dart\
   search\_page.dart
3. P0-3 首页高频列表图片链路优化。\
   文件：\
   home\_page.dart\
   user\_avatar.dart\
   cached\_image.dart\
   要求：将首页卡片中的 Image.network 与 NetworkImage 改为统一缓存组件链路（优先复用 CachedImage / UserAvatar）。\
   要求：为列表图片指定 memCacheWidth/memCacheHeight（按渲染尺寸估算，避免原图进内存）。\
   要求：首图仍保持可点击预览行为不变。
4. P0-4 首页卡片减负（低风险）。\
   文件：\
   home\_page.dart\
   要求：降低 boxShadow 复杂度（减小 blur/alpha 或改轻量边框方案），保留视觉层级。\
   要求：不允许删除核心信息区块（用户、标题、摘要、操作栏）。
5. P1-1 修复 LazyLoadImage 为真实“滚动期延迟加载”。\
   文件：\
   cached\_image.dart\
   要求：移除当前“拿到 renderObject 就直接加载”的逻辑。\
   要求：使用 Flutter 原生 Scrollable.recommendDeferredLoadingForContext(context) 进行滚动期延迟。\
   要求：滚动中展示轻占位，停止滚动后自动进入图片加载。\
   要求：禁止引入 visibility\_detector。
6. P1-2 长列表 delegate 参数统一。\
   文件：\
   home\_page.dart\
   ai\_news\_list\_view\_v2.dart\
   post\_reply\_list.dart\
   要求：统一 cacheExtent 策略（建议基于视口高度倍率，而非固定魔法值）。\
   要求：按场景配置 addAutomaticKeepAlives/addRepaintBoundaries/addSemanticIndexes。\
   要求：避免无意义 keepAlive 导致内存增长。
7. P1-3 高频列表页 Image.network/NetworkImage 二轮替换。\
   范围优先：用户关注/粉丝/浏览历史/搜索结果/主题列表。\
   要求：只处理长列表内高频头像与缩略图，不触碰详情页大图逻辑。

**5. 验收标准（DoD）**

1. 功能验收。\
   下拉刷新、上拉加载、图片显示、图片预览、进入详情、评论/回复操作行为与改造前一致。
2. 性能验收。\
   首页连续快速滑动 15 秒，主观无明显卡顿。\
   Profile 模式下长列表场景平均 FPS 目标 >= 55。\
   长列表场景中 >16ms 帧占比明显下降（相对基线至少改善 20%）。\
   图片缓存峰值稳定，无明显 OOM 风险增长。
3. 代码验收。\
   flutter analyze 通过。\
   flutter test 通过。\
   新增/改动代码有必要注释，且无重复实现。
